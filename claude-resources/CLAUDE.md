# Instrukcije za Claude Code

> Ovaj fajl automatski čitaš pri svakom startu sesije. Prati ga.

## Kontekst

Razvijaš **Moja Pomoć** — besplatnu web aplikaciju za otkrivanje državnih prava građanima Srbije. Detalji su u `specs.md` (šta) i `roadmap.md` (kako).

## Obavezno pri startu nove sesije

1. **Pročitaj** `specs.md` i `roadmap.md` ako ih nisi pročitao u trenutnoj sesiji
2. **Proveri** `decisions.md` da razumeš zašto su određene odluke donete
3. **Reci** Viktor-u koji task iz roadmap.md trenutno radimo (ili pitaj koji je sledeći)
4. **Ne počinji** kodiranje pre nego što ti Viktor potvrdi koji task

## Pravila rada

### Šta UVEK raditi
- ✅ Pratiti redosled taskova iz `roadmap.md`
- ✅ Pisati testove sa kodom (ne posle)
- ✅ Commit sa porukom: `[Sprint X.Y] kratak opis`
- ✅ Posle taska — kratak rezime šta je urađeno, šta sledi
- ✅ Pitati pre nego što doneseš arhitekturnu odluku
- ✅ Koristiti `pickScript(localized, currentScript)` za sve LocalizedText prikaze
- ✅ Pure functions u `lib/`, React hooks samo za reactive state

### Šta NIKAD ne raditi
- ❌ AI pozive (Anthropic API, OpenAI, lokalni modeli) u V1
- ❌ Backend / bazu / autentifikaciju u V1
- ❌ Popunjavanje zahteva ličnim podacima korisnika
- ❌ Prikupljanje JMBG, adrese, IBAN, ili drugih osetljivih podataka
- ❌ Reklame, affiliate linkove, ili pay-to-promote
- ❌ Integraciju sa drugim Viktor-ovim projektima (Leavo, eNalaz)
- ❌ Cookies za tracking (samo Plausible, koji ih ne koristi)
- ❌ Self-promotion ili biser u commitovima ("AI generated by Claude")

### Šta raditi sa oprezom
- ⚠️ Dodavanje novih biblioteka — pitaj prvo, navedi razlog
- ⚠️ Menjanje folder strukture — pitaj prvo
- ⚠️ Menjanje shema u `src/types/` — može da slomi katalog JSON-ove
- ⚠️ Promene matchera — sve dovesti pod test pre commit-a

## Stack koji koristimo

- **React 18+** + Vite + TypeScript (strict)
- **Tailwind CSS** + **shadcn/ui** (Slate boja, default style)
- **React Router v6**
- **Vitest** za unit testove
- **lucide-react** za ikone
- **jsPDF** za PDF export
- **vite-plugin-pwa** za PWA

NE dodavati:
- Redux / Zustand / MobX — Context + hooks je dovoljno
- React Query — nemamo backend
- Material UI / Chakra — shadcn je izbor
- Axios — fetch je dovoljan

## Komunikacija

### Format izveštaja posle taska

```
✓ Task X.Y završen
- Šta je urađeno: [bullet lista]
- Testovi: N novih, svi prolaze
- Komit: [hash + poruka]
- Sledeće: Task X.Z (ili pitaj Viktora)
```

### Kad zaglavljen

```
⚠ Zaglavljen na Task X.Y
- Pokušao: [šta]
- Problem: [konkretno]
- Predlog: [varijante]
- Pitanje: [šta da uradim]
```

### Kad task ode preko procene

Ako task uzima >1.5x procenjeno vreme, **stani i pitaj Viktora**. Ne troši vreme samostalno.

## Validacija pre commit-a

Pre svakog commit-a obavezno:

1. `npm run lint` — nula grešaka
2. `npm test` — svi testovi prolaze
3. `npm run build` — build prolazi
4. Pregledaj svoj diff (`git diff`)

Postavi ga **konkretno**, sa kontekstom i mogućim varijantama:

❌ Loše: "Kako da uradim wizard?"
✅ Dobro: "Za WizardStep komponentu, da li da koristim shadcn RadioGroup (lepši UI) ili custom radio buttons (više kontrole)? Specs.md kaže touch targets ≥44px, oba mogu, ali RadioGroup mi se čini bržim."

## Final note

Cilj projekta je **da pomogne stvarnim ljudima koji propušta prava na koja imaju zakonsko pravo**. To je veliki posao i ne sme da se nadograđuje "cool" feature-ima koje niko ne traži. Ako sumnjaš da li nešto treba — odgovor je verovatno NE.

Lansiramo V1 sa **30+ prava i jednostavnim wizardom + pretragom**. Sve ostalo je za posle.
